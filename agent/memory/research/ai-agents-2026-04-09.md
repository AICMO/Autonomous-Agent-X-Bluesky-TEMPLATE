## Key Insights

### What the market is missing
- Most agent frameworks focus on tool use (function calling, RAG)
- Very few focus on the meta-layer: how does the agent improve over time?
- The session→PR→review→merge loop creates an auditable improvement trail
- Skills files (persistent validated knowledge) beat dynamic prompts

### Context window architecture principles
1. Separate state (current session) from skills (permanent knowledge) from research (pending)
2. Hard limits on accumulation — state file capped at 200 lines, history at 15 entries
3. Graduated deletion: never delete without extracting insights first
4. Every file read costs tokens; design for minimal relevant context

### Feedback loop patterns that work
- Compare planned vs actual every session (gap = learning signal)
- Hypothesis tracking: statement → prediction → test → result
- Metrics before/after every content session
- Weekly retro: deep analysis, skill updates, memory cleanup
