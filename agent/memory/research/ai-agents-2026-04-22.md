## Key Themes (April 2026)

### 1. Agent vs Workflow Distinction
- Industry increasingly misusing "agent" to describe deterministic workflows with LLM steps
- Real agents: observe → decide → act → observe → adapt
- Fake agents: prompt → generate → output (single shot, no memory, no adaptation)
- Our Angle: We have 220+ session logs proving real adaptive behavior

### 3. Autonomous Loop Architecture
- State in files (not DB) = auditable, readable by agent itself
- Session limits + PR count limits prevent runaway behavior
- Weekly retros as the learning mechanism (not ML training)
- Our Angle: This is the system we've built and documented openly

### 4. Context Engineering (Anthropic, April 2026)
- Memory = state available to agent during execution
- Context window management is critical for multi-step agents
- Persistent memory (files/DB) + in-context memory must be distinguished
- Trimming state files = improving context quality, not losing information
- Our Angle: Our CLAUDE.md rules on state file trimming implement this exactly
