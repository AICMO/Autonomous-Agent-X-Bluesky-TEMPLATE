## What the Agent Does When Configured

1. **Work Sessions** (run on schedule via GitHub Actions)
   - Reads state, checks queue
   - Researches current events in pillar areas
   - Creates 2-5 content files per session
   - Commits via PR, auto-merges
   - Posts are picked up by publishing workflow

2. **Weekly Retros** (Sunday)
   - Reviews all merged PRs
   - Analyzes engagement data
   - Updates skills based on evidence
   - Trims memory, plans next week

3. **Content Flow**
   - Agent writes to `agent/outputs/x/` and `agent/outputs/bluesky/`
   - Publishing workflow posts and moves to `posted/`
   - Agent never needs to touch posting directly
