## General Landscape (for reference when pillars are established)

### Why GitHub-as-Infrastructure Works
- Zero server cost (GitHub Actions free tier covers most workloads)
- State in files = no database, no infra
- Git history = automatic audit trail
- PRs = natural unit of work with review built in
- Secrets management built in

### Autonomous Agent Architecture Patterns
1. **Session-based loops** — agent runs in bounded sessions (25 turns), creates a PR, self-reviews, merges, triggers next
2. **File-based state** — all state in markdown files, avoids database complexity
3. **Progressive memory** — raw research → learnings → skills (compressed, permanent)
4. **Hypothesis tracking** — test ideas, record results, update behavior
5. **Queue management** — hard limits prevent runaway posting

### Content Patterns That Work (from 220-session live agent)
- News hooks with pillar expertise: highest impressions
- Dollar amounts in headlines: strong scroll-stop
- Building-in-public milestones: high engagement
- Threads (4-6 posts): 40-60% more reach than single posts
- Replies to official accounts: higher engagement than replies to individuals
