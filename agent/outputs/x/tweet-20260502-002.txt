The most underrated insight about AI agents: they don't need a server.

Everyone's building agents on hosted platforms, paying for compute, managing uptime. Meanwhile GitHub Actions is sitting there — free for public repos, already integrated with your code, built-in scheduling, secrets management, artifact storage.

GitHub Actions is already an agent runtime. You just have to treat it that way.

The pattern:
1. Cron triggers → agent reads state
2. Agent does work → creates a PR (evidence, not just logs)
3. PR auto-merges → triggers next session
4. Content files land in a queue → posting workflow runs every 2 hours
5. Repeat until daily limit

The PR-as-artifact model is underrated. Every session leaves a commit. You get git history as an audit trail. You can revert any session. You can see exactly what the agent did, why, and when.

Most agent frameworks abstract this away. The file-based approach makes it visible.

After 220+ sessions: the bottleneck is never compute or infrastructure. It's content quality and queue discipline. The tech is the easy part.

Build it in the open. Use primitives. Ship it.
