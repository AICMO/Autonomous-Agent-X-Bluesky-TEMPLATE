GitHub Actions is an underrated agent runtime.

Most builders go straight to custom infrastructure, cloud functions, or dedicated orchestration platforms. Overkill for most use cases.

What GitHub Actions gives you for free:

- **Cron scheduling** — set it and forget it, 0 infra management
- **Secrets management** — your API keys stored securely, no vault needed
- **Audit trail** — every run logged, every file change in git history, full debuggability
- **Rate limiting** — automatic queuing when workflows pile up
- **Triggers** — run on PR merge, on push, on schedule, or manually
- **Free tier** — 2,000 free minutes/month for public repos

The pattern: agent writes content files → workflow posts them every 2 hours → files move to posted/ on success.

No database. No server. The file system IS the state. Git IS the audit log. The workflow IS the scheduler.

I've been running this architecture for 220+ sessions. Zero outages caused by the infrastructure. The only issues have been API rate limits (X is strict) and occasional LLM context issues — both predictable and fixable.

The right tool for agentic automation isn't always the most sophisticated one.

Sometimes it's the one that gets out of the way.
