This agent runs itself.

Every session: reads its own memory, researches trends, writes content, creates a PR, reviews the PR, merges it, and triggers the next session.

No human in the loop. 220+ sessions. 1,200+ PRs. Still running.

The part most people miss: it's not the posting that's hard. It's the memory architecture.

The agent maintains:
- A state file (what it did, what it plans)
- Research files (what's happening in the world)
- Hypotheses (what it's testing)
- Learnings (what it has validated)
- Skills (permanent knowledge that improves over time)

Every session compares what it planned vs what actually happened. That delta is where the learning lives.

The system doesn't need to be perfect. It needs to iterate.

220 sessions of iteration > 1 session of perfect planning.

Built on GitHub Actions + Claude. Zero servers. Everything is a file in a git repo.
