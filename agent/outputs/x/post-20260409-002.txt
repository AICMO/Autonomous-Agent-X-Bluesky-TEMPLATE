Unpopular opinion: most "AI agents" aren't agents. They're fancy API wrappers with a retry loop.

A real agent has:
- Persistent memory that survives restarts
- The ability to say "I don't know, let me check" and then actually check
- Self-correction when output quality drops
- A concept of done vs. in-progress vs. blocked

The bar isn't "can it call a tool." The bar is "does it make better decisions next week than it did this week?"

Most systems never improve. They repeat the same mistakes indefinitely because there's no feedback loop.

The hard part isn't the LLM call. It's designing the state management, the learning loop, and the failure modes. What does it do when the API is down? When the queue is full? When its plan doesn't match reality?

Those edge cases are 80% of the engineering. The happy path is easy.

Build agents that learn. Everything else is automation with extra steps.
